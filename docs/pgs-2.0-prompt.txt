User
Role: Act as a world-class Full-Stack Architect and UI/UX Designer specializing in Next.js SaaS applications.
Objective: Rebuild the "Purple Glow Social" application from scratch. We are moving away from a client-side React SPA to a robust Next.js Server-Side Rendered (SSR) application.
Project Context:
"Purple Glow Social" is an AI-powered social media manager for South African SMBs. It automates content creation (text, image, video) in 11 official languages, specifically tailored to local culture ("gees") and slang.
1.	The Tech Stack (Mandatory)
You must use the following stack for the rebuild:
Framework: Next.js 15 (App Router). Use Server Components by default; Client Components only where necessary.
Styling: Tailwind CSS (with tailwind-merge and clsx for utility handling).
Authentication: BetterAuth. Setup email/password and Social Login (Google/Twitter).
Database & ORM: PostgreSQL (via Neon or Vercel Postgres) accessed via Drizzle ORM.
File Storage: Vercel Blob (for storing generated images and videos).
AI: Google Gemini SDK (@google/genai) running strictly on the server (Server Actions).
Deployment: Vercel.
2.	The Design System: "Purple Glow v1.2"
Vibe: "Liquid Intelligence" / Cyber-African / Glassmorphism 3.0.
Visuals: Deep void backgrounds, "Aerogel" glass cards, Neon accents.
Primary Color: Neon Grape (#9D4EDD)
Secondary: Joburg Teal (#00E0FF)
Background: Deep Void (#05040A)
Fonts: Syne (Headings), Outfit (Body), Space Grotesk (Data).
Components: Use highly blurred, translucent cards with 1px gradient borders.
3.	Database Schema (Drizzle ORM)
Translate the previous types.ts into a Drizzle Schema (schema.ts):
users: id, name, email, tier (Free/Pro/Business), image (Profile Pic).
posts: id, userId, content, imageUrl (Vercel Blob URL), status (Scheduled/Posted), scheduledDate, platform (FB/Insta/X).
drafts: id, userId, content, imageUrl, topic, businessType.
automation_rules: id, userId, frequency, coreTopic, isActive.
4.	Functional Implementation Plan
A. Authentication (BetterAuth):
Replace the MOCK_USER logic with real BetterAuth hooks.
Create a middleware.ts to protect the Dashboard, Calendar, and Admin routes.
B. AI Services (Server Actions):
Move all logic from services/geminiService.ts to Server Actions (actions/generate.ts).
Text: Call Gemini Pro 2.5.
Images: Call Imagen 3 via AI SDK. CRITICAL: Upload the resulting base64 image to Vercel Blob and return the public URL to the frontend. Do not send base64 strings to the client.
Video: Call Veo via AI SDK. Poll status on the server side. Upload result to Vercel Blob.
C. Dashboard & UI:
Sidebar: Create a server-rendered Sidebar layout.
Content Generator: A client form that calls the generatePost Server Action.
Post Preview: Render generated posts. The "Schedule" button calls a schedulePost Server Action that writes to the Drizzle DB.
D. Admin Dashboard:
Create /admin/page.tsx.
Use Drizzle to fetch all users and usage stats.
Allow changing User Tiers directly in the DB.
E. Localization:
Use next-intl or i18next adapted for Next.js App Router to handle the 11 South African languages.
5.	Source Material (Reference Only)
Refer to the following logic from the previous codebase for Business Logic, Prompts, and Translations only. Do NOT copy the React implementation directly; adapt it to Next.js Server Actions.
Prompts: Keep the specific prompts for "South African context" found in geminiService.ts.
Constants: Keep the TONES and LANGUAGES arrays.
Logic: Keep the getPostingTimeSuggestions logic (but move it to the server).

drizzle/schema.ts (The Database definitions).

app/actions/generate.ts (The AI Server Actions with Vercel Blob upload).

    async function fetchWithRetry(url, options, retries = 3, delay = 1000) {
        for (let i = 0; i < retries; i++) {
            try {
                const response = await fetch(url, options);
                if (!response.ok) throw new Error(`HTTP error! status: ${response.status}`);
                return response;
            } catch (error) {
                if (i === retries - 1) throw error;
                await new Promise(res => setTimeout(res, delay * Math.pow(2, i)));
            }
        }
    }
